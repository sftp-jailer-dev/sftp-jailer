<objective>
Add a per-user log-detail modal (M-USER-LOG) accessible from S-USERS via uppercase L. The modal shows the user's last 20 login attempts plus tier counters (success / targeted / noise / unmatched) for the time window read from config.LockdownProposalWindowDays.

Per D-01: uppercase L on S-USERS opens M-USER-LOG directly. Lowercase d (S-USER-DETAIL) and uppercase D (M-DELETE-USER-FW-RULES) are unchanged - regression guards.

Per D-02: 5-column row layout matches S-LOGS wide-mode (ts UTC | user | source IP | tier glyph | raw excerpt). OSC 52 copies the focused row.

Per D-03: header strip renders LOG-06 tier counters via Queries.PerUserBreakdown. Per Pitfall 5 the existing PerUserBreakdown signature MUST be extended to take a sinceNs filter so the header counts match the window stated in the strip - otherwise the displayed numbers would be lifetime counts, not last-N-days. The signature change is non-breaking: existing callers pass 0 to disable the filter (mirrors FilterEvents.SinceNs convention).

Per D-04: time window read from config.LockdownProposalWindowDays (default 90, range [1, 3650]). Avoids config-knob proliferation.

Per D-05: empty state copy "No login attempts for `<user>` in the last `<N>` days."

Per Architectural Responsibility Map: M-USER-LOG is a new screen package internal/tui/screens/userlog/ - one screen per dir. Data layer untouched aside from the PerUserBreakdown signature extension.

Purpose: closes TUI-10 polish gap; admin sees per-user observation slice without leaving S-USERS.
Output: 1 new screen package (model + tests); 1 query signature extension; 1 keybind dispatch + factory seam in users; 1 bootstrap wire in main.go.

Per D-18: this is one of 4 independent plans (one per requirement); plans are parallelizable in any order.
Per D-22: every task uses atomic commit-per-step (RED test commit -> GREEN implementation commit -> SUMMARY commit at plan close).

      package userlog

      // M-USER-LOG modal: per-user observation slice. Reads from
      // store.Queries.PerUserBreakdown (header tier counters) and
      // FilterEvents (last 20 raw rows). Time window comes from
      // config.LockdownProposalWindowDays per TUI-10 D-04.

      type Model struct {
          username   string
          q          *store.Queries
          settings   *config.Settings
          windowDays int
          breakdown  store.UserBreakdown
          events     []store.Event
          cursor     int
          err        error
          loaded     bool
          width      int
          height     int
      }

      func New(username string, q *store.Queries, settings *config.Settings) *Model {
          return &Model{
              username:   username,
              q:          q,
              settings:   settings,
              windowDays: settings.LockdownProposalWindowDays,
          }
      }

      // Init dispatches both queries in parallel via tea.Batch.
      func (m *Model) Init() tea.Cmd {
          return tea.Batch(m.loadBreakdown(), m.loadEvents())
      }

      func (m *Model) loadBreakdown() tea.Cmd {
          return func() tea.Msg {
              sinceNs := time.Now().Add(-time.Duration(m.windowDays)*24*time.Hour).UnixNano()
              ub, err := m.q.PerUserBreakdown(context.Background(), m.username, sinceNs)
              return loadedBreakdownMsg{ub: ub, err: err}
          }
      }

      func (m *Model) loadEvents() tea.Cmd {
          return func() tea.Msg {
              sinceNs := time.Now().Add(-time.Duration(m.windowDays)*24*time.Hour).UnixNano()
              evs, err := m.q.FilterEvents(context.Background(), store.FilterOpts{
                  User:    m.username,
                  SinceNs: sinceNs,
                  Limit:   20,
              })
              return loadedEventsMsg{events: evs, err: err}
          }
      }
