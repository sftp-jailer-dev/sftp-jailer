<objective>
Add password_aging_days and password_stale_days as editable rows in S-SETTINGS using the existing inline-edit + textinput pattern (D-11). Reuse config.Validate for range + strict-ordering checks (D-12). Persist via config.Save for round-trip (D-13).

Tighten config.Validate with the documented [1, 3650] upper bound (the existing implementation enforces < 1 and strict-ordering aging < stale but does NOT enforce the documented max=3650 cap). This puts the bound in config.Validate per the research recommendation - single source of truth.
